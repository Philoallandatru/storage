# KV Cache 跨厂商基准测试 — K4 结果（8B 高并发）
**日期：** 2026-06-10
**负载：** K4 — LLaMA-3.1-8B KV cache，16 个并发用户，120 秒，BurstGPT 轨迹回放
**目标：** 揭示高并发小模型 KV cache 压力下跨厂商存储层性能。

配套报告：`kv-cache-4disk-K5-headline-2026-06-10.md`（70B，4 用户）。

---

## 概要

**Biwin X570 以决定性优势胜出。** K4（高并发 8B）进一步拉大了 K5 中已见的差距：

| 指标 | Biwin vs. WD SN570 | Biwin vs. ZhiTai Ti600 | Biwin vs. Seagate FC530 |
|---|---:|---:|---:|
| 读取吞吐量 | **+103 %** | **+28 %** | **+34 %** |
| 读取 P99 延迟 | **−74 %** | −14 % | **−57 %** |
| 读取 P999 尾部延迟 | **−74 %** | −35 % | **−56 %** |
| 写入 P99 延迟 | **−85 %** | **−88 %** | −38 % |
| 处理的条目数（2 分钟） | **+113 %** | +28 % | +33 % |

**关键跨场景发现：** 在 K4 中，**ZhiTai Ti600 在读取带宽、P99 和 IOPS 上均超过 Seagate FC530**。K5（70B）中观察到的 Phison E18 优势在小模型高并发压力下发生逆转。Seagate 是唯一在不同场景间相对排名发生变化的厂商。

---

## 测试方法

### 负载（K4）
- **模型：** LLaMA-3.1-8B（8 卡张量并行）
- **并发用户数：** 16
- **持续时间：** 120 秒
- **轨迹：** BurstGPT 请求到达模式，使用 `--trace-speedup 1000` 回放
- **生成模式：** `none`
- **缓存配置：** `--gpu-mem-gb 0 --cpu-mem-gb 0`（强制走 NVMe 路径）

K4 与 K5 在两个压力维度上不同：
- 更多并发用户（16 vs. 4）→ 更高的 IOPS 需求
- 更小的模型 → 每个条目的 KV 占用更小，每 GB 条目数更多

这用于测试各磁盘是在 IOPS 饱和之前还是带宽饱和之前达到瓶颈。

### 被测磁盘

| Vendor ID | Model | 层级 / 定位 | 挂载点 |
|---|---|---|---|
| biwin_x570 | BIWIN X570 1 TB | 主流（带 DRAM，此前测试对象） | `/run/media/ficus/新加卷` |
| seagate_fc530 | Seagate ZP1000GV30012 | 高端（Phison E18） | `/mnt/ai_ssd2` |
| zhitai_ti600 | ZHITAI Ti600 1 TB | 国产（YMTC NAND） | `/mnt/ai_ssd1` |
| wd_sn570 | WDC WDS960G2G0G | 入门级（无 DRAM） | `/mnt/ai_ssd0` |

测试**串行**执行，各磁盘使用相同的 kv-cache.py 参数，seed=42，replay-cycles=0。

Biwin 在同样的 2 分钟窗口内处理的条目数比 WD **多 113%**。

## 各磁盘评估

### 🥇 Biwin X570 — 推荐
- 32K 读取 IOPS，3.14 GB/s 读取带宽，73 ms P99 — 全面最优。
- 14 ms 的读取 P50 延迟表明，在持续 16 用户并发读取下，控制器/DRAM 组合能够从容满足需求。

### 🥈 ZhiTai Ti600 — 对小模型负载可接受
- 2.46 GB/s 读取带宽表现扎实；84 ms 的 P99 在许多服务场景下可接受。
- **注意事项：** 325 ms 的写入 P99 仍偏高——纯 KV cache 服务场景下可接受，但对于混合检查点+推理场景有风险。

### 🥉 Seagate FC530 — 信号混杂
- 37 ms 的读取 P50 延迟（除 WD 外最高）表明 Phison E18 控制器在 16 用户并发下提前遇到了瓶颈。
- 对于低并发 70B（K5）合理，但对于高并发 8B 并非最佳选择。

### 4️⃣ WD SN570 — 不推荐
- 64 ms 的读取 P50 延迟意味着磁盘每次读取都在排队。
- 281 ms 的读取 P99 意味着磁盘偶尔停顿达 300 ms；用户会感受到明显的卡顿。
- 应避免用于任何 KV cache 卸载负载。

## K4 vs K5 跨场景观察

相同四块磁盘，两种截然不同的压力模式：

| Disk | K5 read BW (GB/s) | K4 read BW (GB/s) | Δ | K5 entries | K4 entries | Δ |
|---|---:|---:|---:|---:|---:|---:|
| Biwin X570 | 2.77 | **3.14** | +13 % | 1695 | **3259** | +92 % |
| ZhiTai Ti600 | 1.93 | 2.46 | +27 % | 1264 | 2551 | +102 % |
| Seagate FC530 | 2.09 | 2.34 | +12 % | 1281 | 2450 | +91 % |
| WD SN570 | 1.49 | 1.55 | +4 % | 910 | 1527 | +68 % |

**观察结果：**
- 四块磁盘在 K4 中处理的条目数均约为 2 倍（每次 KV 条目占用更小）。
- Biwin 在带宽上扩展最好（+13%），条目数增长最多（+92%）。
- **WD 带宽几乎无改善（+4%）**——无 DRAM 控制器是瓶颈，与负载无关。
- **Seagate 在 K4 中排名下降**（K5 中第 2，现为第 3）。Phison E18 偏好大块顺序读取；K4 的较小条目大小+更高并发暴露了此弱点。
- ZhiTai 在 K4 中的相对增益（+27% 带宽）证实 YMTC NAND 处理小条目 IO 优于大条目 IO。

### P99 延迟：K4 vs K5

| Disk | K5 read P99 (ms) | K4 read P99 (ms) | Δ |
|---|---:|---:|---:|
| Biwin X570 | 93.8 | 72.9 | −22 % |
| ZhiTai Ti600 | 212.1 | 84.4 | −60 % |
| Seagate FC530 | 131.2 | 169.3 | +29 % |
| WD SN570 | 240.0 | 281.2 | +17 % |

Biwin 和 WD 在 K4 中变快（更小的条目→更少排队）。**Seagate 和 ZhiTai 朝着相反方向变化**，证实它们各有不同的最佳工作点。

---

## 注意事项

- **120 秒时间较短。** K4 未测试 GC/写入放大悬崖。30 分钟的持续运行才能暴露性能随时间下降的磁盘。
- **所有层强制走 NVMe。** 实际部署中即使有少量 HBM 层（≥8 GB）也会减少磁盘压力。厂商间的相对排名预计不变，但绝对值会下降。
- **纯 KV cache。** 无检查点、嵌入查询、RAG。真实的混合负载会暴露额外的写入尾部惩罚。
- **Seed=42 可复现性。** 之前的两次 K4 运行（v1 扫描+本次）结果分别为 3256→3259 个条目（0.1% 差异）。四块磁盘均在各自的噪声范围内。
