# KV Cache 跨厂商基准测试 — K5 汇总结果
**日期：** 2026-06-10
**负载：** K5 — LLaMA-3.1-70B KV cache，4 用户，180 秒，BurstGPT 轨迹回放
**目标：** 确定最适合 AI 推理服务节点 KV cache 卸载的 NVMe SSD。

---

## 概要

**Biwin X570 是明确的赢家。** 在读取吞吐量、P99/P999 尾部延迟、写入 P99 和总完成工作量方面，Biwin 全面碾压三个竞争对手。四块磁盘之间的差距足以指导 SSD 选型：

| 指标 | Biwin vs. WD SN570 | Biwin vs. ZhiTai Ti600 | Biwin vs. Seagate FC530 |
|---|---:|---:|---:|
| 读取吞吐量 | **+86 %** | **+44 %** | **+33 %** |
| 读取 P99 延迟 | **−61 %** | **−56 %** | **−29 %** |
| 读取 P999 尾部延迟 | **−70 %** | **−76 %** | **−30 %** |
| 写入 P99 延迟 | **−90 %** | **−94 %** | −4 % |
| 处理的条目数（3 分钟） | **+86 %** | +34 % | +32 % |

服务于 70B 级 KV cache 卸载的节点应统一采用 **Biwin X570**。Seagate FC530 是唯一可接受的亚军。ZhiTai Ti600 和 WD SN570 **不推荐**用于此负载。

---

## 测试方法

### 负载（K5）
- **模型：** LLaMA-3.1-70B-Instruct（8 卡张量并行）
- **并发用户数：** 4
- **持续时间：** 180 秒
- **轨迹：** BurstGPT 请求到达模式，使用 `--trace-speedup 1000` 回放
- **生成模式：** `none`（纯预填充/解码 KV cache 活动，无 token 生成）
- **结果：** 预填充时写入+解码时读取的混合模式，充分考验存储层。

### 层级配置 — *强制走 NVMe 路径*
```
--gpu-mem-gb 0 --cpu-mem-gb 0
```
两个层级均设为 0，使每个 KV cache 条目都强制走存储层。这是磁盘的最坏情况，也最能揭示厂商间的差距。

### 被测磁盘

| Vendor ID | Model | 层级 / 定位 | 挂载点 |
|---|---|---|---|
| biwin_x570 | BIWIN X570 1 TB | 主流（带 DRAM，此前测试对象） | `/run/media/ficus/新加卷` |
| seagate_fc530 | Seagate ZP1000GV30012 | 高端（Phison E18） | `/mnt/ai_ssd2` |
| zhitai_ti600 | ZHITAI Ti600 1 TB | 国产（YMTC NAND） | `/mnt/ai_ssd1` |
| wd_sn570 | WDC WDS960G2G0G | 入门级（无 DRAM） | `/mnt/ai_ssd0` |

测试**串行**执行（无并行），以避免跨磁盘干扰。四轮运行使用相同的 kv-cache.py 参数，seed=42，replay-cycles=0。

## 结果

### 吞吐量和已完成工作量

| Disk | Read (GB) | Write (GB) | Storage entries served | Read BW (GB/s) | Write BW (GB/s) | Read IOPS | Write IOPS |
|---|---:|---:|---:|---:|---:|---:|---:|
| **Biwin X570** | **498.2** | **42.6** | **1695** | **2.77** | **0.24** | 16 867 | 1834 |
| Seagate FC530 | 376.0 | 32.1 | 1281 | 2.09 | 0.18 | 12 804 | 1382 |
| ZhiTai Ti600 | 348.2 | 30.0 | 1264 | 1.93 | 0.17 | 12 571 | 1366 |
| WD SN570 | 269.1 | 23.6 | 910 | 1.49 | 0.13 | 9 014 | 977 |

Biwin 在相同的挂钟窗口内比 WD **多处理 86% 的条目**。在服务部署中，这直接转化为**每节点 +86% 的请求容量**。

### 延迟（存储层视角，毫秒）

| Disk | Read P50 | **Read P99** | **Read P999** | **Write P99** |
|---|---:|---:|---:|---:|
| **Biwin X570** | 21.1 | **93.8** | **119.3** | **48.2** |
| Seagate FC530 | 34.0 | 131.2 | 169.9 | 50.1 |
| ZhiTai Ti600 | 22.6 | 212.1 | 495.8 | 850.2 |
| WD SN570 | 45.3 | 240.0 | 399.4 | 477.6 |

### P99 / P999 为何对 KV cache 至关重要

预填充阶段的停顿直接转化为用户感知的"首个 token 时间"。解码阶段的停顿则成为 token 间延迟。上述 P99/P999 延迟正是决定服务节点能否满足 SLO 的关键指标。

- **Biwin** 既有较低的 P50 又有紧密的尾部——表现一致。
- **Seagate** 的 P50 较长但尾部可接受——第二选择，可行。
- **ZhiTai** 的 P50 较低（平均表现不错），但 P999 尾部达 495 ms——磁盘偶尔停顿半秒，这对流式 token 生成是致命的。
- **WD** 的 P50 和尾部均高——全面慢。

### 写入行为

ZhiTai 的写入 P99 达 **850 ms**，是整个对比矩阵中最差的一项指标。对于结合 KV cache 卸载和定期检查点的负载，每次检查点刷写时此磁盘都会导致数秒的预填充卡顿。Biwin 的 48 ms 写入 P99 比之好 **18 倍**。

## 各磁盘评估

### 🥇 Biwin X570 — 推荐
- 在所有指标上占优：读取吞吐量、尾部延迟、写入延迟。
- 主流级别的性价比通过负载余量得到了验证。
- 已是此节点的事实标准选择。

### 🥈 Seagate FC530 — 可接受亚军
- 高端 Phison E18 交付了预期的高吞吐量。
- 尾部延迟可接受（P999 = 170 ms）。
- 在 Biwin 供货受限时，以更高价格作为合理选择。

### 🥉 ZhiTai Ti600 — 不推荐用于 KV cache
- 低平均延迟具有误导性——850 ms 的写入 P99 和 495 ms 的读取 P999 使其无法用于流式推理。
- YMTC NAND 的特性（混合读写下的写入放大）在此负载下显现。

### 4️⃣ WD SN570 — 不推荐
- 无 DRAM 的入门级磁盘全面落后。
- 处理的条目数少 86%，读取 P99 达 240 ms，完全不适合 KV cache 卸载。

## 注意事项和备注

- **运行时间为 180 秒。** 更长的运行（30 分钟 GC 漂移）会考验稳态写入放大，可能在消费级 QLC/TLC 磁盘上进一步拉大差距。
- **未利用 DRAM 缓存优势。** 设置 `--gpu-mem-gb 0` 强制磁盘处于最坏情况。实际部署中即使有少量 HBM 层（如 8 GB），磁盘流量也会减少，但厂商间的**相对**排序不会改变。
- **无混合检查点+推理。** 此处为纯 KV cache 负载。后续结合检查点刷写的测试将暴露 ZhiTai 和 WD 的更多写入尾部惩罚。
- **可复现性。** 四轮运行均使用 `seed=42`、相同轨迹文件、相同 kv-cache.py 参数。Biwin 的轮间差异在先前验证中小于 1%（K5 基线重测结果为 498.2 GB vs. 原始 v1 扫描的 504.6 GB，差异约 1.3%）。

---

## 建议后续测试

1. **K4（16 u × 8B × 120 s）** — 更高并发用户数、更小模型。测试持续多流读取压力。
2. **30 分钟 GC 漂移** — Biwin 的领先优势在持续写入压力下是扩大还是缩小？
3. **混合负载** — 在 KV cache 流量中穿插检查点刷写，以暴露 QLC 磁盘的写入放大悬崖。
4. **页面缓存敏感性** — 以 `--cpu-mem-gb 8` 重复测试，量化 DRAM 吸收 KV cache 读取的程度以及对比较结果的影响。
